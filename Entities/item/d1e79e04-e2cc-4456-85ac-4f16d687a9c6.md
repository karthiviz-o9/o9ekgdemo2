# Data Dictionary: Item (Supply Chain Planning)

## Overview
The `Item` (or SKU/Material) is the core master data entity in supply chain planning. It represents any physical good or material that is procured, manufactured, stored, or sold by the organization.

## Entity Details
* **Entity Name:** `SupplyChain_Item`
* **Primary Key:** `Item_ID`
* **Description:** Contains all static and planning-related master data for materials across the supply chain network.

---

## Attribute Dictionary

### 1. Core Identification
| Attribute Name | Data Type | Constraint | Description | Example Values |
| :--- | :--- | :--- | :--- | :--- |
| **Item_ID** | String(50) | Primary Key, Not Null | Unique identifier for the item (SKU or Material Number). | `RM-10045`, `FG-9921` |
| **Item_Description** | String(255) | Not Null | Detailed text description or name of the item. | `Aluminium Sheet 2mm` |
| **Base_UOM** | String(10) | Not Null | The primary Unit of Measure used for storing and tracking. | `EA`, `KG`, `L` |
| **Status** | String(20) | Not Null | The current lifecycle state of the item. | `Active`, `Phase-Out` |

### 2. Classification & Categorization
| Attribute Name | Data Type | Constraint | Description | Example Values |
| :--- | :--- | :--- | :--- | :--- |
| **Material_Type** | String(30) | Not Null | Categorizes the item based on its role in the supply process. | `Raw Material`, `Intermediate Material`, `Finished Good` |
| **Item_Class** | String(10) | Optional | ABC classification code used for inventory segmentation. | `A`, `B`, `C` |
| **Product_Family** | String(50) | Optional | Higher-level grouping used for aggregate planning (S&OP). | `Electronics`, `Beverages` |

### 3. Supply Chain Planning Parameters
| Attribute Name | Data Type | Constraint | Description | Example Values |
| :--- | :--- | :--- | :--- | :--- |
| **Is_Planning_Item** | Boolean | Not Null | Flag indicating if this item should be included in planning runs. | `TRUE`, `FALSE` |
| **Make_Buy_Code** | String(10) | Not Null | Indicates whether the item is manufactured internally or purchased. | `Make`, `Buy` |
| **Lead_Time_Days** | Integer | Not Null | Total time required to procure or manufacture the item. | `14`, `30` |
| **Safety_Stock_Qty** | Decimal(10,2) | Optional | Minimum buffer inventory held to protect against volatility. | `500.00` |
| **Min_Order_Qty** | Decimal(10,2) | Optional | Minimum Order Quantity (MOQ) allowed by the supplier/plant. | `100.00` |

### 4. Physical & Financial Attributes
| Attribute Name | Data Type | Constraint | Description | Example Values |
| :--- | :--- | :--- | :--- | :--- |
| **Unit_Weight** | Decimal(10,4) | Optional | The weight of a single base unit (for logistics calculations). | `2.5000` |
| **Weight_UOM** | String(10) | Optional | The unit of measure for the `Unit_Weight`. | `KG`, `LBS` |
| **Standard_Cost** | Decimal(12,4) | Not Null | The baseline cost used for inventory valuation. | `14.5000` |
