# Functional Specification Document: Create Network (Rust)

## 1. Overview & Purpose
**Product Name:** Create Network (Rust Crate)
**Purpose:** To ingest flat, relational master data entities (Items, Locations, Lanes, BOMs, Routings, Resources) and compile them into a unified, strongly-typed directed graph in memory. This graph will serve as the static physical foundation for a downstream supply chain heuristic solver (e.g., MRP engine, inventory optimization) to execute against.

## 2. Scope
* **In Scope:** Reading master data inputs, validating referential integrity via strict typing, building nodes (SKU-Locations) and edges (Lanes/BOMs), calculating aggregated network properties, and exposing a solver-ready graph structure.
* **Out of Scope:** Ingesting transactional data (Demand, Inventory) or executing the actual solver logic. This crate solely acts as the topology generator.

---

## 3. System Architecture & Core Crates

### 3.1. Inputs (Data Sources)
The system will accept tabular data conforming to the defined data dictionaries. Data ingestion will utilize:
* **`serde` & `serde_derive`:** For strict deserialization of incoming data into Rust structs.
* **`csv` or `polars`:** For reading flat files or processing large data frames efficiently.
* **`sqlx` (Optional):** If reading directly from a relational database.

### 3.2. Processing Engine (Graph Construction)
* **`petgraph`:** The standard Rust graph data structure library. We will use a `DiGraph` (Directed Graph) to model the network flow.
* **`rayon`:** For parallelizing data validation and initial data transformations before graph insertion.

---

## 4. Functional Requirements

### 4.1. Data Ingestion & Strict Validation
* **REQ-1.1:** The crate must deserialize all input tables into strictly typed Rust `struct`s.
* **REQ-1.2 (Referential Integrity Check):** The system must validate foreign keys before graph insertion. Missing references must return a standard Rust `Result::Err` to gracefully handle bad master data.
* **REQ-1.3 (Cycle Detection):** The builder must use `petgraph::algo::is_cyclic_directed` to ensure no infinite loops exist in BOMs or distribution lanes.

### 4.2. Node Generation (The "What" and "Where")
* **REQ-2.1:** Create a unique Node for every valid combination of `Item_ID` + `Location_ID` (SKU-Location). 
* **REQ-2.2:** Each node in the `petgraph` will store a `NodeData` struct containing static constraints like `safety_stock_qty` and `storage_capacity`.

### 4.3. Edge Generation (The "How" and "Movement")
The system must connect the nodes using a Rust `enum` to handle the two distinct types of edges safely:
* **REQ-3.1 (Transportation Edges):** Draw directed edges from `Source` to `Destination`. Attach a `Transport` variant containing `transit_time_days`, `mode`, and `co2_emission_factor`.
* **REQ-3.2 (Transformation Edges):** Draw directed edges from `Component` to `Parent_Item`. Attach a `Transform` variant containing `quantity_per_assembly` and `resource_id`.

### 4.4. Network Property Aggregation
* **REQ-4.1 (Cumulative Lead Time):** Traverse the directed graph backwards from Finished Goods to calculate the longest path (critical path) lead time.
* **REQ-4.2 (Cost Rollup):** Calculate baseline standard costs recursively down the graph, factoring in BOM multipliers and transportation rates.

---

## 5. The Output Model (Rust Data Structures)

The final output passed to the heuristic solver will be a `petgraph` structure utilizing defined structs and enums. 

```rust
use petgraph::graph::{DiGraph, NodeIndex};
use serde::{Serialize, Deserialize};

// The core node representing an Item at a specific Location
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct SkuLocationNode {
    pub item_id: String,
    pub location_id: String,
    pub lead_time_offset: u32,
    pub safety_stock: f64,
}

// Enum to handle the two different ways inventory moves through the network
#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum EdgeType {
    Transport {
        lane_id: String,
        transit_time: u32,
        cost: f64,
    },
    Transform {
        bom_id: String,
        quantity_multiplier: f64,
        resource_required: String,
    },
}

// The final solver-ready network model
pub struct SupplyChainNetwork {
    // petgraph Directed Graph holding our custom structs
    pub graph: DiGraph<SkuLocationNode, EdgeType>,
    // A fast-lookup hashmap mapping "Item@Location" string keys to petgraph NodeIndexes
    pub node_map: std::collections::HashMap<String, NodeIndex>, 
}
