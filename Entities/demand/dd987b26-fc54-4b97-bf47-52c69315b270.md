# Data Dictionary: Demand (Requirements & Forecasts)

## Overview
The `Demand` entity represents the outward-facing requirements placed upon the supply chain. It captures both firm commitments (Sales Orders) and statistical expectations (Forecasts). This is the primary driver that triggers the planning engine to generate the Supply Plan.

## Entity Details
* **Entity Name:** `Demand`
* **Primary Key:** `Demand_ID`
* **Description:** Contains all prioritized requirements from customers and internal planning (like safety stock targets) across the planning horizon.

---

## Attribute Dictionary

### 1. Core Identification & Source
| Attribute Name | Data Type | Constraint | Description | Example Values |
| :--- | :--- | :--- | :--- | :--- |
| **Demand_ID** | String(50) | Primary Key, Not Null | Unique identifier for the specific demand requirement. | `SO-45992`, `FCST-MAR-01` |
| **Demand_Type** | String(30) | Not Null | Categorizes the source of the requirement. | `Sales Order`, `Forecast`, `Safety Stock`, `Promotional` |
| **Customer_ID** | String(50) | Foreign Key, Optional | The specific customer or account requesting the items. | `WALMART-001`, `AMAZON-PRIME` |
| **Priority** | Integer | Not Null | The importance of this demand (1 = Highest). Used for "fair share" logic during stock shortages. | `1`, `10`, `99` |

### 2. Product & Geography
| Attribute Name | Data Type | Constraint | Description | Example Values |
| :--- | :--- | :--- | :--- | :--- |
| **Item_ID** | String(50) | Foreign Key, Not Null | The specific item requested (links to `SupplyChain_Item`). | `FG-9921` |
| **Location_ID** | String(50) | Foreign Key, Not Null | The warehouse or node where the demand is expected to be fulfilled from. | `DC-US-EAST` |

### 3. Quantity & Timing
| Attribute Name | Data Type | Constraint | Description | Example Values |
| :--- | :--- | :--- | :--- | :--- |
| **Quantity** | Decimal(10,2) | Not Null | The total number of units required. | `1200.00`, `50.00` |
| **UOM** | String(10) | Not Null | The unit of measure for the demand quantity. | `EA`, `PALLET`, `KG` |
| **Demand_Date** | Date | Foreign Key, Not Null | The date the customer wants the product or the date the forecast is bucketed into. | `2026-06-01` |
| **Probability_Pct** | Decimal(5,2) | Optional | For Forecasts, the confidence level of the demand materializing. | `0.85` (85%), `1.00` (Firm) |

### 4. Status & Sales Data
| Attribute Name | Data Type | Constraint | Description | Example Values |
| :--- | :--- | :--- | :--- | :--- |
| **Status** | String(20) | Not Null | The lifecycle status of the demand line. | `Open`, `Partially Filled`, `Forecast` |
| **Sales_Price** | Decimal(12,4) | Optional | The unit price agreed upon or expected for this demand. | `199.99`, `15.50` |
| **Currency** | String(3) | Not Null | The currency code for the sales price. | `USD`, `EUR` |
