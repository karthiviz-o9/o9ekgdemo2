# Data Dictionary: Actuals (Sales, Shipments, Production)

## Overview
The `Actuals` entity acts as a consolidated ledger of all completed, historical events across the supply chain. It records past sales demand, fulfilled shipments, and completed manufacturing runs to provide a baseline for future forecasting and performance analytics.

## Entity Details
* **Entity Name:** `Actuals`
* **Primary Key:** `Transaction_ID`
* **Description:** Contains transactional records of past supply chain activities used to feed statistical forecasting engines and operational reporting.

---

## Attribute Dictionary

| Attribute Name | Data Type | Constraint | Description | Example Values |
| :--- | :--- | :--- | :--- | :--- |
| **Transaction_ID** | String(50) | Primary Key, Not Null | Unique identifier for the historical ledger entry. | `ACT-100492`, `SHP-8821` |
| **Activity_Type** | String(30) | Not Null | Categorizes the nature of the historical event. | `Sales`, `Shipment`, `Production` |
| **Reference_Document**| String(50) | Optional | The original system document number that triggered the actuals. | `SO-9948`, `WO-2011` |
| **Date** | Date | Foreign Key, Not Null | The exact calendar date the activity occurred (links to `Dim_Enterprise_Time`). | `2025-11-24` |
| **Item_ID** | String(50) | Foreign Key, Not Null | The specific material that was sold, shipped, or produced (links to `SupplyChain_Item`). | `FG-9921` |
| **Location** | String(50) | Foreign Key, Not Null | The node where the activity took place (e.g., shipping DC, selling store, or manufacturing plant). | `DC-US-WEST`, `PLANT-004` |
| **Quantity** | Decimal(10,2) | Not Null | The physical amount of the item involved in the transaction. | `150.00`, `5000.00` |
| **UOM** | String(10) | Not Null | The unit of measure for the transaction quantity. | `EA`, `KG`, `PALLET` |
| **Financial_Value** | Decimal(12,2) | Optional | The recorded revenue (for sales) or cost (for production/shipments) of the transaction. | `14500.00`, `250.50` |
