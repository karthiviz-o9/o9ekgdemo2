# Data Dictionary: Location (Supply Chain & Planning)

## Overview
The `Location` entity represents any physical or logical node in the supply chain network. This includes manufacturing plants, distribution centers (DCs), warehouses, retail stores, and sometimes specific zones within a larger facility. It governs where inventory resides and the capacity constraints of that node.

## Entity Details
* **Entity Name:** `Location`
* **Primary Key:** `Location_ID`
* **Description:** Contains master data for physical addresses, operational capacities, and network hierarchies used in supply chain planning and record-keeping.

---

## Attribute Dictionary

### 1. Core Identification & Types
| Attribute Name | Data Type | Constraint | Description | Example Values |
| :--- | :--- | :--- | :--- | :--- |
| **Location_ID** | String(50) | Primary Key, Not Null | Unique identifier for the site or facility. | `DC-US-EAST`, `PLANT-004` |
| **Location_Name** | String(150) | Not Null | The operational name or description of the facility. | `Atlanta Main Hub`, `Berlin Assembly` |
| **Location_Type** | String(50) | Not Null | Categorizes the primary function of the location within the network. | `Warehouse`, `Manufacturing`, `Retail Store`, `3PL` |
| **Status** | String(20) | Not Null | The current operational status of the location. | `Active`, `Under Construction`, `Closed` |

### 2. Geographical & Address Data
| Attribute Name | Data Type | Constraint | Description | Example Values |
| :--- | :--- | :--- | :--- | :--- |
| **Address_Line_1** | String(255) | Not Null | Primary physical street address. | `123 Logistics Blvd` |
| **City** | String(100) | Not Null | The city or municipality where the location resides. | `Atlanta`, `Frankfurt` |
| **State_Province** | String(50) | Optional | The state, province, or regional identifier. | `GA`, `Hessen` |
| **Country_Code** | String(3) | Not Null | Standardized ISO 3-letter country code. | `USA`, `DEU`, `IND` |
| **Postal_Code** | String(20) | Not Null | The zip or postal routing code. | `30301`, `60311` |
| **Latitude** | Decimal(9,6) | Optional | Geo-coordinate for precise mapping and routing calculations. | `33.748995` |
| **Longitude** | Decimal(9,6) | Optional | Geo-coordinate for precise mapping and routing calculations. | `-84.387982` |

### 3. Capacity & Planning Metrics
| Attribute Name | Data Type | Constraint | Description | Example Values |
| :--- | :--- | :--- | :--- | :--- |
| **Storage_Capacity** | Decimal(12,2) | Optional | The maximum allowable inventory volume or space at this location. | `500000.00`, `15000.00` |
| **Capacity_UOM** | String(10) | Optional | The unit of measure for storage capacity. | `Pallets`, `SQFT`, `CBM` |
| **Storage_Condition** | String(50) | Optional | The environmental constraints of the facility (vital for perishables/chemicals). | `Ambient`, `Cold Storage`, `Hazmat` |
| **Max_Daily_Throughput**| Integer | Optional | The maximum number of units or orders the facility can process (receive/ship) in one day. | `10000`, `500` |

### 4. Operations & Network Hierarchy
| Attribute Name | Data Type | Constraint | Description | Example Values |
