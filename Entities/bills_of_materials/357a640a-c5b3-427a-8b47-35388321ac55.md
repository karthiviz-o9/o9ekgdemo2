# Data Dictionary: Bills of Materials (BOM)

## Overview
The `Bills of Materials` (BOM) entity defines the manufacturing recipe or hierarchical structure of a product. It links a parent assembly to the child components and raw materials required to build it.

## Entity Details
* **Entity Name:** `Bills_Of_Materials`
* **Primary Key:** `BOM_ID` 
* **Description:** Contains the core structural data used by Material Requirements Planning (MRP) to explode demand from finished goods down to raw materials.

---

## Attribute Dictionary

| Attribute Name | Data Type | Constraint | Description | Example Values |
| :--- | :--- | :--- | :--- | :--- |
| **BOM_ID** | String(50) | Primary Key, Not Null | Unique identifier for this specific component line within the BOM. | `BOM-1049-010` |
| **Parent_Item** | String(50) | Foreign Key, Not Null | The finished good or sub-assembly being manufactured (links to `SupplyChain_Item`). | `FG-9921` |
| **Component_Item** | String(50) | Foreign Key, Not Null | The raw material or part being consumed to build the parent (links to `SupplyChain_Item`). | `RM-10045` |
| **Component_UOM** | String(10) | Not Null | The unit of measure for the component (must align with the component's base UOM). | `EA`, `KG` |
