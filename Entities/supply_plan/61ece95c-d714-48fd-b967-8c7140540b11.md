# Data Dictionary: Integrated Supply Plan (Procure, Produce, Distribute, Fulfill)

## Overview
The `Integrated Supply Plan` entity is the central output of the planning engine. It consolidates all future-looking planned orders—what needs to be bought (Procurement), made (Production), moved (Distribution), and sold (Fulfillment)—into a single unified ledger. 

## Entity Details
* **Entity Name:** `Supply_Plan`
* **Primary Key:** `Plan_ID`
* **Description:** Contains all system-generated or user-firmed planned activities across the end-to-end network, linking demand to its corresponding supply.

---

## Attribute Dictionary

### 1. Order Identification & Type
| Attribute Name | Data Type | Constraint | Description | Example Values |
| :--- | :--- | :--- | :--- | :--- |
| **Plan_Order_ID** | String(50) | Primary Key, Not Null | Unique identifier for the planned activity. | `PLN-PRC-102`, `PLN-MFG-994` |
| **Activity_Type** | String(30) | Not Null | Defines the specific nature of the planned plan. | `Procurement`, `Production`, `Distribution`, `Fulfillment` |
| **Order_Status** | String(20) | Not Null | The current lifecycle state of the planned order. | `System Suggested`, `Firmed by Planner`, `Released` |

### 2. What and Where (Network Linkages)
| Attribute Name | Data Type | Constraint | Description | Example Values |
| :--- | :--- | :--- | :--- | :--- |
| **Item_ID** | String(50) | Foreign Key, Not Null | The specific item being planned (links to `SupplyChain_Item`). | `FG-9921`, `RM-10045` |
| **Source** | String(50) | Foreign Key, Optional | Where the item originates (Vendor for Procure, Plant for Produce/Distribute). | `VEND-001`, `PLANT-004` |
| **Destination** | String(50) | Foreign Key, Not Null | Where the item is going (Plant for Procure, DC for Distribute, Customer for Fulfill). | `DC-US-WEST`, `CUST-992` |
| **Lane_ID** | String(50) | Foreign Key, Optional | The specific routing lane or path assigned to the planned order (links to `Network_Distribution_Lane`). | `LANE-10944`, `BOD-US-01` |

### 3. Quantities & Financials
| Attribute Name | Data Type | Constraint | Description | Example Values |
| :--- | :--- | :--- | :--- | :--- |
| **Planned_Quantity** | Decimal(10,2) | Not Null | The total volume to be procured, produced, moved, or fulfilled. | `5000.00`, `250.00` |
| **UOM** | String(10) | Not Null | The unit of measure for the planned quantity. | `EA`, `PALLET`, `KG` |
| **Estimated_Cost** | Decimal(12,2) | Optional | The projected cost of executing this planned order (material, freight, or manufacturing). | `12500.00`, `450.75` |

### 4. Scheduling & Timeline (Links to `Dim_Enterprise_Time`)
| Attribute Name | Data Type | Constraint | Description | Example Values |
| :--- | :--- | :--- | :--- | :--- |
| **Planned_Start_Date**| Date | Not Null | When the activity must begin (e.g., PO issue date, production start, ship date). | `2026-05-01` |
| **Planned_End_Date** | Date | Not Null | When the activity must finish (e.g., PO receipt, production completion, delivery). | `2026-05-15` |
| **Lead_Time_Offset** | Integer | Optional | The total days between the Start Date and End Date. | `14`, `3` |
